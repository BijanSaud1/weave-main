out the directory skeleton, and dropped .gitkeep files into the empty
folders so they'd actually get tracked. The MCP Python SDK installed
cleanly with pip install mcp. Ollama took a few minutes to pull Llama
3 8B, but once it was down a quick test prompt confirmed it was working.
Finished by installing three of Anthropic's reference MCP servers —
filesystem, fetch, and memory — and starting each one to verify they
came up cleanly.

## Found
The arXiv digest for cs.AI looks like it'll be a firehose — fifty to a
hundred papers a day. cs.CR is more manageable. Going to give the email
digest two weeks; if cs.AI volume becomes unsustainable I'll switch to
RSS feeds and read in a dedicated reader. The CITATION.cff workflow is
genuinely well-thought-out — GitHub picks up changes within a minute,
no extra config needed.

Llama 3 8B runs fast enough on this laptop for prototype work, which
means I probably don't need a bigger model for the early experiments.
The MCP Python SDK has a small, clean API — readable in an afternoon.
Ollama exposes an OpenAI-compatible HTTP API on localhost:11434, which
will make plugging the local model into the agent loop straightforward
later. The reference MCP servers are Node-based and print their tools
list on startup, so I can sanity-check them without writing any code yet.
